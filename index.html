Multi-slide form (pt 1)

Forms can be created without the traditional 'form' tag and can span multiple slides. The resulting values can either be POST-ed to a defined action (a la the standard form attribute) or the values can form a JSON object (is that redundant?) for API use or for custom controllers.

Multiple forms can be applied to a single slide show by utilizing the "data-submit-form" attribute in both the input element and the button element that is responsible for submitting the form.

Multi-slide form (pt 2)

Step-by-step wizards can be created this way, and validation libraries can be added as normal.

Table of Contents

To display buttons leading to every slide, create a block element with the class defined in the conf.json file. The default is "table-of-contents".

Simple-slides will use the string value of certain attributes to add text to the button in this hierarchical order: "data-title-slide" -> "name" -> "id"

Define the class for every button within the table of contents element with the "data-button-class" attribute (i.e., one class for all of the slides' buttons).

Hardcoded buttons
