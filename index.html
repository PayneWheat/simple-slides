Multi-slide form (pt 1)

Forms can be created without the traditional form tag and can span multiple slides. The resulting values can either be POST-ed to a defined action (a la the standard form attribute) or the values can form a JSON object (is that redundant?) for API use or custom controllers.

Multiple forms can be applied to a single slide show by utilizing the "data-submit-form" attribute in both the input element and the button element that is responsible for submitting the form. Their values must match for the form data to be populated into a single object. "form1" in this example.

Table of Contents

To display buttons leading to every slide, create a block element with the class defined in the conf.json file. The default is "table-of-contents".

Simple-slides will use the string value of certain attributes to add text to the button in this hierarchical order: "data-title-slide" -> "name" -> "id"

Add a class value for every button within the table of contents with the "data-button-class" attribute (i.e., one class for all of the slides' buttons).

Hardcoded buttons

Custom buttons can be added to any slide, and the "data-jump-to" attribute is used to specify the id of the slide the button jumps to.

Timed slides

Automatic transitions to the next sequential slide can be enabled with the "data-timed-slide" attribute.

The value of this attribute can be a floating point number or integer, representing the number of seconds before transitioning.

Circular configuration

The slideshow can be configured to be circular, i.e. the last slide will transition back to the first slide. This can be accomplished with the data-jump-to attribute.

This is particularly useful for sliders.

Otherwise by default, no next button will be rendered.
